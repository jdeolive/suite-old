      General

      Why can't I start the OpenGeo Suite?

      The most common reason why the OpenGeo Suite is unable to start is because of a port conflict. By default, the OpenGeo Suite web applications run on port 8080 of the host machine, and if another service using that port, the Suite will not run. To change this port, go to the Preferences page, change the Primary Port, click Save, then restart the OpenGeo Suite. If you are still unable to start the service, you may want to check the Logs for other errors. If you are still stuck, please visit the OpenGeo Suite Community Forum and describe your issue and the steps you've taken to resolve it.

      How do I start/stop the OpenGeo Suite?

      There are a few ways to start the OpenGeo Suite. The simplest way is through the Dashboard, by clicking the Start/Stop button on the bottom right. You can also manually start and stop the suite from the command line by typing "opengeo-suite start" or "opengeo-suite stop" in the root of your application directory.

      How do I make sense of the logs?

      Many Java errors are very long and span multiple lines (all usually being with the word "at ..."). Usually the first line of the error is where the human-readable content is (and it usually is the line that does not contain the "at ..."). You may have to scroll up to find the error you're looking for.

      PostGIS

      Is PostGIS a part of the OpenGeo Suite?

      Yes. This software includes PostGIS, for a complete geospatial software stack.

      Return to top

      How do I know PostGIS is running?

      Since PostGIS is part of the OpenGeo, if the OpenGeo Suite is online then PostGIS will be as well. Check the Dashboard for the status of the OpenGeo Suite.

      Return to top

      How can view/edit my PostGIS database?

      The OpenGeo Suite includes PgAdmin, a desktop graphical database management application. Click on the Manage PostGIS link in the Dashboard to run PgAdmin.

      Return to top

      What port is PostGIS running on?

      By default, PostGIS runs on port 54321. This value can be changes on the Preferences page.

      Return to top

      Can I run the OpenGeo Suite if I already have PostGIS installed?

      Yes. By deafult, PostGIS installations run on port 5432, but the PostGIS database included with the OpenGeo Suite runs on port 54321. As long as these port numbers are different, there should be no conflicts.

      The OpenGeo Suite User Manual has links to methods of finding the projection. If you are unable to determine the projection, you may still be able to use the Layer Importer and load the shapefiles directly into GeoServer, as GeoServer intelligently can determine the projection.

      GeoServer

      Does the GeoServer support connections with ArcSDE or Oracle Spatial databases?

      Yes, however there are certain required files that must be manuall copied from your own ArcSDE/Oracle installation. The files required for ArcSDE are jsde*.jar and 

      jpe*.jar
      from your ArcSDE installation. The files required for Oracle Spatial is ojdbc*.jar from your Oracle installation. Copy the required file(s) into the installation directory under "webapps/geoserver/WEB-INF/lib" and restart. You should then see ArcSDE/Oracle as an option in GeoServer.

      Return to top

      Where is my GeoServer data directory?

      Your GeoServer data directory is located inside your user's home directory at ".opengeo/data_dir".

      Return to top

      Why did I get a Java stack trace with GeoServer in my web browser?

      Java stack traces can happen for many reasons, ranging from incorrect requests to software bugs. The first lines of the stack trace normally have the most helpful information. If you are unable to discern the particular error you're receiving, please the entire error into a problem request at the OpenGeo Suite Community Forum. Please describe what you were doing immediately prior to the error occurring, as well as your operating system, software version, and other relevant information.

      Return to top

      How do I add lots of data to GeoServer at once?

      The OpenGeo Suite includes a Layer Importer as an extension to GeoServer that allows for batch loading of PostGIS database tables and shapefiles.

      Return to top

      Can I run the OpenGeo Suite if I already have GeoServer installed?

      Yes, however GeoServer runs on port 8080 by default. If you already have another GeoServer instance running on port 8080, the OpenGeo Suite will not start properly, so you will need to change the port that GeoServer runs on. To change this port, go to the Preferences page, change the Primary Port, click Save, then restart the OpenGeo Suite.

      GeoExplorer

      How do I increment the bookmark number in GeoExplorer?

      The bookmark number increments automatically when you compose a new map.

      Why is my OpenLayers map blank (or pink)?

      Blank/pink maps in OpenLayers are usually caused by either a JavaScript error, or an invalid connection to a web service. You may wish to look at the Error Console for your browser, or even better, use a JavaScript debugger. If using Mozilla Firefox, a popular debugger is Firebug.

      Return to top

      Styler

      Does Styler include all of the functionality contained in the SLD spec?

      Styler includes the most commonly used tags contained in the SLD specification, but does not attain 100% compatibility. Certain extras, such as GeoServer vendor options, are not supported or preserved. If you are unsure about editing an existing SLD file with Styler, please make a backup first.

      Return to top

      Why are my vendor options are not rendered when I view an existing SLD in Styler?

      Styler does not recognize or preserve GeoServer vendor options, which are not part of the SLD specification. If you are unsure about editing an existing SLD file with Styler, please make a backup first.

      Return to top

      Why is Styler blank, with no layers in the layer pane?

      This is often caused by a corrupt layer entry in GeoServer. You may wish to delete and reload any recent layers or layer groups. Also, the Logs may have more detailed information.
