Attribute-based Polygon

This example styles the polygons differently based on the "pop" (Population) attribute.
Each polygon in our fictional country has a population that is represented by the population ("pop") attribute. This style contains three rules that alter the fill based on the value of "pop" attribute, with smaller values yielding a lighter color and larger values yielding a darker color.

The three rules are designed as follows:

Rule order Rule name / type Population ("pop") Color
1 SmallPop Less than 200,000 #66FF66
2 MediumPop 200,000 to 500,000 #33CC33
3 LargePop Greater than 500,000 #009900

The order of the rules does not matter in this case, since each shape is only rendered by a single rule.

The first rule, on lines 7-21, specifies the styling of polygons whose population attribute is less than 200,000. Lines 10-15 set this filter, with lines 11-14 setting the "less than" filter, line 12 denoting the attribute ("pop"), and line 13 the value of 200,000. The color of the polygon fill is set to a light green (#66FF66) on line 18.

The second rule, on lines 22-42, is similar, specifying a style for polygons whose population attribute is greater than or equal to 200,000 but less than 500,000. The filter is set on lines 25-36. This filter is longer than in the first rule because two criteria need to be specified instead of one: a "greater than or equal to" and a "less than" filter. Notice the And on line 26 and line 35. This mandates that both filters need to be true for the rule to be applicable. The color of the polygon fill is set to a medium green on (#33CC33) on line 39.

The third rule, on lines 43-57, specifies a style for polygons whose population attribute is greater than or equal to 500,000. The filter is set on lines 46-51. The color of the polygon fill is the only other difference in this rule, which is set to a dark green (#009900) on line 54.
