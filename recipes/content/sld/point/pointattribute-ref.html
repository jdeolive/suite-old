Attribute-based Point Documentation

This example alters the size of the symbol based on the value of the population ("pop") attribute.
This style contains three rules. Each <Rule> varies the style based on the value of the population ("pop") attribute for each point, with smaller values yielding a smaller circle, and larger values yielding a larger circle.

The three rules are designed as follows:

Rule order Rule name / type Scale denominator Point size
1 SmallPop Less than 50,000 12
2 MediumPop 50,000 to 100,000 12
3 LargePop Greater than 100,000 16

The order of the rules does not matter in this case, since each shape is only rendered by a single rule.

The first rule, on lines 7-27, specifies the styling of those points whose population attribute is less than 50,000. Lines 10-15 set this filter, with lines 6-9 setting the "less than" filter, line 12 denoting the attribute ("pop"), and line 13 the value of 50,000. The symbol is a circle (line 19), the color is dark blue (#0033CC, on line 21), and the size is 8 pixels in diameter (line 24).

The second rule, on lines 28-54, specifies a style for points whose population attribute is greater than or equal to 50,000 and less than 100,000. The population filter is set on lines 31-42. This filter is longer than in the first rule because two criteria need to be specified instead of one: a "greater than or equal to" and a "less than" filter. Notice the ``And`` on line 32 and line 41. This mandates that both filters need to be true for the rule to be applicable. The size of the graphic is set to 12 pixels on line 51. All other styling directives are identical to the first rule.

The third rule, on lines 55-75, specifies a style for points whose population attribute is greater than or equal to 100,000. The population filter is set on lines 58-63, and the only other difference is the size of the circle, which in this rule (line 72) is 16 pixels.

The result of this style is that cities with larger populations have larger points.
